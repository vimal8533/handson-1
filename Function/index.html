Q4. "Print output: var x = 21; var girl = function () { console.log(x); var x = 20; }; girl ();"

Ans:
        var x = 21;
    var girl = function () {
        console.log(x);
        var x = 20;
    };
    girl ();
     

Output:undefind
Explanation:when the function girl is executed, a local variable x is
created and is hoisted to the top of the function scope. Since the
assignment var x = 20; comes after the console.log(x); statement, x is
undefined at the time of the console.log() statement

Q5. "Print output: var x = 21; girl (); console.log(x) function girl() { console.log(x); var x = 20; }; "

Ans:
            var x = 21;
            girl ();
            console.log(x)
            function girl() {
            console.log(x);
            var x = 20;
           };


Output:undefind 
       21
Explanation:This is because of variable hoisting within the girl 
function. When the girl function is invoked before the console.log(x) 
statement outside the function, a local variable x is created within 
the function scope. This local variable x is hoisted to the top of the 
function scope, which means it exists but has an initial value of
 undefined at the time of the console.log(x) statement inside the 
 function.Therefore, the first console.log(x) statement inside the
 girl function will output undefined. After that, the global variable 
 x is accessed outside the girl function, which has a value of 21. 
 So, the second console.log(x) statement will output 21.

Q7. Write a function that accepts parameter n and returns factorial of n

    function fact(n){
        if(n<=1){
            return 1;
        }else{

    }

Q10. "Guess the output var a = 10; (function (){ console.log(a); var a = 20; })();"

Ans:
    var a = 10;
    (function (){
        console.log(a);
        var a = 20;
    })();
Output:undefined
Explanation:due to variable hoisting in JavaScript. The variable a is 
declared and assigned the value 10 in the global scope. Inside the 
immediately-invoked function expression (IIFE), a local variable 
a is also declared with the var keyword.Since variable declarations 
are hoisted to the top of their scope, the var a declaration within 
the IIFE is hoisted to the top of the function. At the time of the 
console.log(a) statement, the local variable a exists but has an initial 
value of undefined because the assignment var a = 20 has not yet been
reached.
Therefore, when console.log(a) is executed, it output undefined.
